# 途风旅游预订平台（cn.toursforfun.com）产品需求文档（PRD）
版本：V1.0　日期：2026-07-09


## 一、背景与目标
途风（ToursForFun，携程旗下）是面向全球华人的出境游/入境游预订平台，核心业务是跟团游、私家团、小团、当地玩乐、定制旅行、邮轮等标品与半标品旅游产品的在线展示、比价与预订，并配套多渠道客服（电话、微信、WhatsApp、Line）与App/小程序生态。

本次PRD目标： 基于当前官网已呈现出的功能形态，反推并系统化一份产品需求文档，明确用户价值、功能模块边界、验收标准，并给出可执行的分阶段（MVP→成长期→成熟期）实施路线图，供后续迭代与新功能开发（或改版）参考。
### 产品目标
提升海外目的地跟团游/小团/定制游的转化率（从浏览到下单）。
通过多语言、多币种、多渠道客服，覆盖全球华人客群（中国大陆、港澳台、北美、澳新等）。
强化价格透明度与信任背书（价格保障、成团保障、行中体验保障），降低决策门槛。
沉淀私域流量（App、小程序、微信、WhatsApp、Line、订阅邮件）提升复购与LTV。
### 目标用户
出境游用户：计划前往美加欧澳新等地旅游的中国大陆/港澳台/海外华人，偏好中文服务、跟团省心。
入境游用户：海外华人回国或外国人来华旅游（中国入境游产品线）。
高净值定制客户：需要私家团、商务定制、多国深度游的用户。
老年/家庭/亲子客群：偏好省心小团、包接送机、纯中文导游。

## 三、核心功能详细需求（Functional Requirements）
### F1. 全局基础能力
| 功能点 | 需求描述 | 验收标准 |
|---|---|---|
| 多语言站点 | 简体中文/繁体中文/英文三套独立站点，一键切换并保留当前浏览上下文 | 切换语言后停留在对应目的地/品类页面，而非强制跳转首页 |
| 多币种切换 | 支持8种货币展示价格（不涉及实际收单币种可另行确认） | 切换币种后全站价格实时换算，含牌价与汇率更新时间标注 |
| 全局搜索 | 支持目的地/产品关键词搜索 | 搜索结果按相关性+销量排序，支持无结果兜底推荐 |
| 会员体系 | 登录/注册、邀请有礼 | 支持第三方登录（微信/手机号），新用户享注册礼包 |

### F2. 内容导流与导航
| 功能点 | 需求描述 | 验收标准 |
|---|---|---|
| 顶部分类导航 | 美洲/欧洲/澳新/小团/优选/定制/邮轮/高端度假等一级分类 + 二级目的地 | 导航层级不超过3级点击可达产品列表 |
| 首页专题Banner | 支持配置多个可点击轮播/静态Banner，关联UTM追踪参数（如ici/icn） | 运营可在CMS后台自主配置Banner图片、跳转链接、上下线时间，无需工程介入 |
| 热门/小众/当季推荐词云 | 目的地关键词快捷入口，按热度/季节动态调整 | 支持后台配置权重及排序 |
| 目的地卡片矩阵 | 按大区块（美国/加拿大拉美/欧洲/花样世界）展示子目的地入口 + 主打折扣标签 | 每区块支持独立配置折扣角标（如"5折起"） |

### F3. 商品与交易
| 功能点 | 需求描述 | 验收标准 |
|---|---|---|
| 产品列表/筛选 | 按目的地、天数、团型（大团/小团/私家团）、价格区间筛选排序 | 筛选结果1秒内返回，支持多条件组合 |
| 产品详情页 | 展示行程亮点、天数、团型人数上限、赠品、划线价与折扣价、库存/团期日历 | 价格必须实时同步，划线价需展示原价与依据（避免虚假折扣风险） |
| 热销榜单 | 按目的地/品类展示Top产品，支持横向翻页（1/3等分页） | 榜单更新频率≥每日一次，基于真实销量数据 |
| 用户热评 | 展示带头像、评分、日期、精选评论摘要的用户点评，关联具体产品/团期 | 评论需来自真实已完成订单用户（UGC真实性校验） |
| 促销与优惠券 | 买赠（买2送1/送2）、满减、专属折扣码 | 优惠规则需在下单页清晰展示，防止价格误导投诉 |
| 下单与支付 | 选择团期→填写出行人信息→选择增值服务（接送机等）→支付 | 全链路加密支付，支持多种支付方式，下单成功后即时生成订单确认与邮件/短信通知 |

### F4. 信任与服务保障
| 功能点 | 需求描述 | 验收标准 |
|---|---|---|
| 品牌保障体系 | 价格保障、消费透明、旅行安全、成团/行程保障、行中体验保障、隐私安全 8项承诺图标化展示 | 每项承诺需可点击查看详细条款页面，而非纯展示型图标 |
| 客户协议与政策 | 取消和修改条例、信用卡支付验证书、全球入境签证信息 | 政策文本版本化管理，更新需保留历史版本可追溯 |
| 公告/新闻中心 | 时效性信息（如航线取消、政策变化）+ 品牌公关新闻 | 重大公告（如航班取消）需在首页/相关产品页有醒目提示联动 |
| 多渠道客服 | 电话（美国/国际/台湾/香港/大陆5条热线）、微信、WhatsApp、Line、在线反馈 | 7x24小时响应，各渠道SLA需可监控（如平均响应时长） |

### F5. 特色业务线
| 功能点 | 需求描述 | 验收标准 |
|---|---|---|
| 定制旅行 | 案例展示（新西兰、西班牙等）+ "定制师"咨询入口 | 案例页可一键发起定制咨询表单，48小时内定制师响应 |
| 全球邮轮 | 按航线（南北极/中文服务/欧洲内河/长江）分类展示 | 邮轮产品需展示船型、舱型、停靠港，因其决策复杂度高于陆地团 |
| App/小程序 | 二维码下载入口、专属优惠（App端折扣/小程序扫码券） | 需追踪扫码转化率，衡量私域渠道ROI |

### F6. SEO与内容运营
| 功能点 | 需求描述 | 验收标准 |
|---|---|---|
| Footer长尾SEO链接 | 目的地/景点/邮轮航线/热门搜索词长尾页面 | 需配套SEO落地页内容（非纯链接堆砌），避免被判定为低质页面 |
| 目的地指南/资讯 | 独立内容频道，提供签证、天气、当地玩乐攻略 | 内容需定期更新，建立内容日历 |
| 邮件订阅 | 首页/Footer订阅入口，承诺新用户礼包 | 订阅后触发欢迎邮件序列（Welcome Flow），符合CAN-SPAM/PIPEDA等合规要求 |



## 四、非功能性需求
| 类别 | 要求 |
|---|---|
| 性能 | 首页首屏加载 ≤2秒（含大量Banner图，需CDN+WebP压缩，当前已采用_ipx图片处理服务，应延续） |
| 多端适配 | PC/H5/App/小程序体验一致，H5为主要入口（华人用户多来自搜索/微信分享） |
| 国际化合规 | 需满足中国大陆（ICP备案已具备）、加拿大（PIPEDA）、欧盟（GDPR，如面向欧洲用户）等数据合规要求 |
| 安全 | 支付全链路加密（已声明"安全加密支付"），需定期渗透测试与PCI-DSS合规（如自建支付网关） |
| 可运营性 | Banner、专题页、优惠券、热销榜排序等应可由运营人员通过后台配置，不依赖工程发版 |
| 数据与追踪 | 全渠道UTM/ici/icn参数体系需统一，支撑营销归因分析 |



## 五、分阶段实施计划
说明：按照"打好地基→提升转化→精细化运营与生态扩展"的产品成熟度路径分为三个阶段，每阶段建议周1周。
### 阶段一：基础体验闭环
目标： 确保用户能"搜索-浏览-下单-支付"全链路顺畅，建立基本信任背书。

核心导航与目的地分类（美洲/欧洲/澳新/小团/定制/邮轮）
产品列表页 + 筛选（目的地、天数、团型、价格区间）
产品详情页（行程、价格、赠品、团期日历）
下单与支付主流程（含订单确认通知）
多语言（中/繁/英三站）与多币种切换基础版
品牌保障图标展示（8项承诺静态版，先展示后补详情页）
7x24客服热线接入（电话为主）
基础SEO：Footer长尾链接框架搭建

验收里程碑： 用户可完整走完"搜索目的地→选产品→下单支付→收到确认"全链路，且转化数据可被追踪。


### 阶段二：转化提升与信任强化
目标： 提升转化率与客单价，降低决策疑虑，扩大私域触点。

热销榜单模块（按目的地/品类，动态更新）
用户评价/热评展示（含头像、评分、精选摘要，UGC真实性校验机制）
促销引擎：买赠、满减、专属折扣码，支持运营后台自主配置
Banner/专题活动位CMS化（运营可自主上下线，无需工程介入）
品牌保障详情页（价格保障、取消政策、成团保障条款落地）
多渠道客服扩展：微信、WhatsApp、Line、在线反馈接入
App/小程序MVP上线（二维码引流+基础下单能力）
邮件订阅+欢迎邮件序列（Welcome Flow）

验收里程碑： 转化率（浏览→下单）、复访率、客服响应时长等核心指标建立基线并环比提升；用户评价覆盖率达到目标产品的X%以上。


### 阶段三：精细化运营与生态扩展
目标： 深化个性化推荐、私域运营与特色业务线（定制/邮轮），建立长期竞争壁垒。

个性化推荐：基于浏览/历史订单的目的地与产品推荐算法
定制旅行全链路：在线定制咨询表单 + 定制师工作台 + 案例库
全球邮轮专区深化：船型/舱型比价、航线地图可视化
会员体系升级：等级权益、积分、专属折扣、邀请有礼裂变机制
公告/新闻联动机制：重大时效信息（如航班取消）自动触发相关产品页/已购用户的主动提醒
App/小程序深度运营：专属券、扫码转化率追踪、消息推送
数据中台：全渠道UTM归因、营销ROI看板、A/B测试平台
国际合规deep dive：GDPR/PIPEDA专项审计，多地区数据本地化评估

验收里程碑： 私域渠道（App+小程序+微信+WhatsApp+Line）贡献订单占比达到设定目标；定制/邮轮等高客单业务线收入占比提升；建立可持续的A/B测试与数据驱动迭代机制。


## 六、风险与依赖
| 风险 | 说明 | 应对建议 |
|---|---|---|
| 价格展示合规风险 | 划线价/折扣价若无真实原价依据，可能涉嫌虚假宣传 | 建立原价备案机制，价格变动留痕可追溯 |
| UGC评价真实性 | 热评/晒图若非真实订单用户，存在信任风险 | 评价与订单强绑定，仅允许已完成行程用户发表 |
| 多地区合规差异 | 大陆/北美/欧洲用户数据保护法规不同 | 分区域数据存储与隐私政策，法务提前介入 |
| 时效性信息滞后 | 航班取消等公告如更新不及时，影响已购用户体验 | 建立公告-订单联动的主动通知机制（阶段三） |
| 依赖第三方渠道 | 微信/WhatsApp/Line等私域渠道政策变化 | 保持官网/App为核心资产，私域渠道作为增量而非唯一入口 |
